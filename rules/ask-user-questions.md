# AskUserQuestion Convention

When presenting multiple options via AskUserQuestion, follow this convention to speed up decision-making:

## Always Recommend One Option

Mark the best-fit option with **(recommended)** in its text. Example:
- "Standard planning (recommended)"
- "Competing architectures"

## Explain Trade-offs

For each option, include a brief pro/con or context note so the user can make an informed choice without asking follow-ups. Give extra detail on the recommended option — why it fits the current situation.

## Justify the Recommendation

Add a 1-sentence justification after the options explaining WHY the recommended option is the best fit for the current context. Base it on project state, task complexity, risk level, or prior [LEARN] entries — not generic advice.

## When to Skip

- Single-option confirmations (yes/no) — no recommendation needed
- When the user has already stated their preference — just confirm, don't re-recommend
- When all options are equally valid and context-dependent — present them neutrally and say so
